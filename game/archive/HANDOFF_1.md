# The Pilgrim's Path — Design Handoff Document
**For: Opus continuation session**
**Context: This document continues a design conversation. The attached lore.json is canonical. The lore viewer artifact is a working React component. This document captures game design decisions, open questions, and generator architecture so the conversation can continue without loss.**

---

## What We Are Building

Two things, in sequence:

1. **The Pilgrim's Path** — a browser-based static branching narrative. The player witnesses the portal opening at a warehouse rave, makes choices, collects transcendentals and artifacts, and arrives at one of four terminal outcomes. This is a lore delivery mechanism wearing game clothes. It publishes to a static host (Netlify or GitHub Pages) within roughly two days.

2. **The Meta-Tool** — a Claude-powered authoring environment (React artifact using the Anthropic API) that generates candidate story nodes, lets the author accept/reject/edit them, and exports a complete `nodes.json`. This is the generator that builds the game. It is built first; the game is its output.

**The event context:** DJ OOR is performing at a gala in approximately two weeks. The game should be live before then so attendees can explore the lore in advance. Future events will add layers — the architecture should support this.

---

## Core Design Philosophy

**Tight spine, wide edges.** One reliable main path delivers the essential lore. Variability lives in texture — conditional text, artifact color, arrival variants — not in structural divergence. A player who completes the main path understands OOR, Suboont, the transcendentals, and witnesses at least one ritual. Across three playthroughs, a player encounters approximately 80% of the lore.

**Lore delivery is the primary function.** The game is not a horror game. The horror is ambient. The goal is understanding. By the end of a full playthrough, the player should feel what it means to have been part of a collective transcendental harvest — the phenomenology of a great late-night dance session, the Buddhist dissolution of self into crowd, the essence of memory without full recall.

**OOR is never a speaker.** He does not address the player. He is doing a job. The player happens to be in the room. The music is the only communication. OOR gets a separate styled block for actions — clinical, present tense, observational — never dialogue.

**Ignorance is a quiet exit.** Players who make avoidant choices don't get punished. They just fade out — they wake up at home in a kind of dull fugue state with nothing. No fanfare. The game gives them nothing and doesn't explain why.

**The uncertainty is the condition.** Servant identity, portal origin, the nature of Suboont — these are never resolved. The agent generating nodes must treat mystery as a first-class value, not a gap to fill.

## The Data Architecture

Three files. One data format flowing between them.

```
lore.json       → injected into meta-tool as system prompt context
nodes.json      → generated by meta-tool, read by player
artifacts.json  → generated separately, referenced by nodes and player state
```

### Node Schema

```json
{
  "id": "node_014",
  "text": "Base narrative text. Second person. Present tense. Terse.",
  "arrival_variants": [
    { "requires": { "has_tag": "came_through_alley" }, "text": "The cold follows you in from the alley..." },
    { "default": true, "text": "The heat hits before you cross the threshold." }
  ],
  "conditional_text": [
    { "condition": { "has_artifact": "pale_antler" }, "append": "The antler hums against your pocket." },
    { "condition": { "has_transcendental": "unity" }, "append": "You feel the crowd behind you even here." },
    { "condition": { "has_tag": "fearful" }, "append": "Your hands won't stop shaking." }
  ],
  "oor_action": "OOR turns toward the center of the room. It does not look at you. It looks at all of you.",
  "transcendental": "unity",
  "ritual": "circle",
  "temperature": "warm",
  "awards_transcendental": null,
  "awards_artifact": null,
  "perspective": "wanderer",
  "choices": [
    { "text": "Step into the circle", "next": "node_015", "tag": "participates", "requires": null },
    { "text": "Offer the shard", "next": "node_019", "tag": "accepts", "requires": { "has_artifact": "shard_of_resonance" } },
    { "text": "Back away", "next": "node_022", "tag": "withdraws", "requires": null }
  ],
  "terminal": false,
  "outcome": null
}
```

**Key schema notes:**
- `arrival_variants` — the DAG solution. Nodes can be reachable from multiple paths; arrival text varies by tag, body text stays canonical. This gives the "merged location" feel (everyone arrives at the warehouse, but through different doors) without requiring the agent to write context-neutral prose.
- `conditional_text` — appended to base text based on state. Never in base text: "you decided earlier." Always in conditional blocks.
- `oor_action` — separate render block. Italic, offset, lower opacity. Optional per node.
- `temperature` — `cold | cool | warm | hot`. Drives subtle atmospheric text and CSS. Cold = avoidant path. Hot = harvest aligned. Never explained to player.
- `perspective` — `wanderer | familiar | oor`. Filters which nodes are accessible per playthrough.

### State Model

```json
{
  "node": "node_001",
  "perspective": "wanderer",
  "tags": ["witness"],
  "transcendentals": [],
  "artifacts": [],
  "run_artifacts": [],
  "seed": 1713484800000
}
```

- Initialized on page load. 
- `run_artifacts` — seeded at load via timestamp-derived weighted random. 3–4 artifacts drawn from pool per session. Creates variability without requiring save state.
- `tags` — accumulate across choices. Drive conditional text and choice gating.
- `seed` — `Date.now()` at load. Used for all weighted random decisions in session.

### Condition Evaluator

```javascript
function evaluate(condition, state) {
  if (!condition) return true;
  if (condition.has_artifact)       return state.artifacts.includes(condition.has_artifact);
  if (condition.has_transcendental) return state.transcendentals.includes(condition.has_transcendental);
  if (condition.has_tag)            return state.tags.includes(condition.has_tag);
  if (condition.run_has_artifact)   return state.run_artifacts.includes(condition.run_has_artifact);
  return true;
}
```

~20 lines. No framework dependency. Pure function.

### Artifact Pool Schema

```json
{
  "id": "shard_of_resonance",
  "name": "Shard of Resonance",
  "transcendental": "beauty",
  "description": "A fragment of something that vibrated at the moment of creation. It is warm.",
  "effect": "unlocks_choice",
  "weight": 3
}
```

- Pool size: 15–20 artifacts
- Effects: `unlocks_choice | conditional_text | awards_transcendental_on_use`
- Weight 1 = rare, weight 4 = common
- To be generated interactively with the author — not fully pre-specified

### Terminal Outcomes

| ID | Name | Description |
|---|---|---|
| `transcendence` | Transcendence | Fully participated. Portal closes. You carry it home in your body, not your memory. |
| `harvest` | Harvest | Partially participated. Something was taken you did not consciously offer. You feel lighter. You are not sure this is good. |
| `failure` | Failure | Collective experience collapsed. Portal does not close cleanly. Ambient consequences. |
| `escape` | Escape | Withdrew entirely. You are home. You remember nothing. You will wonder. The game gives you nothing. |

## Narrative Structure

```
SPINE (6–8 nodes, hand-authored, linear)
  Portal opening felt / approached
  Arrival at the warehouse
  First awareness of OOR
  OOR's nature becomes apparent
  First choice point: engage or withdraw

BRANCHES (one per transcendental, ~6 nodes each)
  Unity    → the dance circle ritual
  Beauty   → the perfect drop, surrender or guard
  Truth    → OOR looks at you, be seen or deflect
  Goodness → someone near you is frozen, help or focus

CLOSING (4 terminal nodes, one per outcome)
```

Total estimated nodes: 35–45.

**Convergence pattern:** Branches are not fully isolated. Key canonical locations (the dance floor, the portal itself, the closing sequence) are shared nodes with `arrival_variants`. Players on different branches arrive at the same warehouse but through different doors. The location body — the canonical lore — is the same. The arrival text varies by path. This is the DAG resolution: shared canonical moments, variable arrival context.

**Ritual distribution:** A single playthrough witnesses at least one ritual. Three playthroughs covers approximately 80% of the lore. The rituals are not locked to specific branches — they are canonical moments that may be encountered depending on path and run_artifacts.

## Perspectives System

Three playthrough types, unlocked progressively:

| ID | Name | Unlock | Starting Point |
|---|---|---|---|
| `wanderer` | The Wanderer | Always available | Outside, the approach |
| `familiar` | The Familiar | After completing one run | Inside, near OOR |
| `oor` | OOR | Possess holy artifact at game start | The decks, the center |

The OOR perspective is the rarest and most disorienting. Humans as mineral deposits. The music as a tool. The drop as a drill. This is a significant creative undertaking and should be authored last.

The Familiar perspective reframes the mating ritual from the inside — the theremin-bearer and choreographer as player character. Completely different emotional register.

---

## Arc Types

Not all playthroughs follow the same subjective pattern. Two arc types:

**Grounded arc:** Player remains a singular witness throughout. Second-person voice stable. Choices feel personal and consequential. Default experience.

**Dissolution arc:** Unlocked through depth of engagement. Subjective experience destabilizes as a reward. The second-person voice becomes unreliable — the narrator may lose track of who "you" refers to. The primordial plane bleeds into the material. Should feel like something happening to the player, not being explained to them. Accessible without being too esoteric. This is a discovery, not an assignment.

## The Temperature Mechanic

Cold is moral feedback. Never stated as such.

- Exterior/approach nodes: cold, damp, biting
- Avoidant choices: player notices the cold
- Engaged choices: warmth increases
- Dance floor: hot
- Closing ritual: full heat, then stillness

The player may interpret it as the night being cruel and uncaring. It is not the night. It is their decisions leading them away from the heart of Suboont. A second-playthrough player may notice the pattern. The game never confirms it.

Implementation: `temperature` field on nodes drives subtle atmospheric language in `conditional_text` and CSS class on the renderer.

## The Meta-Tool

A React artifact powered by the Claude API. This is the authoring environment — the game is its output.

### What It Does

1. Loads `lore.json` and current `nodes.json` (pasted in or via persistent artifact storage API)
2. Displays a visual branch map — gray (stub), blue (complete), orange (terminal), red (broken reference)
3. Accepts generation parameters
4. Calls Claude API (Opus for generation, Sonnet for quick edits)
5. Returns 2–3 candidate nodes as valid JSON
6. Author picks one, edits inline, commits to graph
7. Exports updated `nodes.json`

### Generation Parameters

- **Branch:** Unity / Beauty / Truth / Goodness / Spine / Terminal
- **Arc position:** Early / Climax / Resolution
- **Arc type:** Grounded / Dissolution
- **Perspective:** Wanderer / Familiar / OOR
- **Player tags coming in:** (selector from tag vocabulary)
- **Previous node ID:** (for narrative continuity)
- **Artifact in play:** (optional)
- **Temperature:** Cold / Warm / Hot
- **Count:** 2 or 3 candidates

### System Prompt Template

```
You are a co-author generating nodes for a branching narrative called The Pilgrim's Path.

LORE CANON (treat as contract, not suggestion):
{lore_json}

VOICE REFERENCE (do not deviate from this register):
{spine_nodes}

FULL CURRENT TREE (for context and continuity):
{nodes_json}

HARD CONSTRAINTS:
- Second person, present tense
- No node may reference a prior choice as fact in base text — use conditional_text blocks
- Terse. Every sentence earns its place.
- OOR is ambivalent, not malevolent. It does not understand humans.
- The transcendentals are minerals to OOR. It is mining.
- Do not resolve ambiguity about portal origin, servant identity, or Suboont's nature
- Temperature field must match the player's current arc alignment
- OOR action blocks: clinical, observational, present tense, never emotional

GENERATE:
Branch: {branch}
Arc position: {position}
Arc type: {arc_type}
Perspective: {perspective}
Player tags coming in: {tags}
Previous node: {previous_node_id}
Artifact potentially in play: {artifact}
Temperature: {temperature}

Return {count} candidate nodes as a valid JSON array matching this schema exactly:
{node_schema}

Return ONLY the JSON array. No preamble. No explanation.
```

## Renderer Notes

- Single HTML file, no framework
- `sessionStorage` optional for back-button survival within a visit
- **No persistence between visits.** State resets on every page load.
- Typography: **DM Mono** for narrative body, **Bebas Neue** for headers and entity names
- Dark background, OOR color palette: orange (#E06010), yellow (#C8A020), violet (#7040D0), pink (#D04870)
- CSS glitch effect on node transitions — keyframe-based, triggered on state change
- Audio triggers on specific node tags (first interaction unlocks audio context):
  - `witnessed_portal` → low resonant hum
  - `oor_action` blocks → subtle distortion
  - Terminal `transcendence` → slow bass swell
  - Terminal `failure` → silence, then a single low tone
  - Terminal `escape` → nothing

## Open Questions / Decisions Not Yet Made

1. **Dice / skill checks** — discussed and tentatively cut for the two-day window. The tag system already creates probabilistic feel via `run_artifacts`. Can be added in a future layer.

2. **Artifact generation** — Blaine wants to do this interactively, generating 8 at a time and accepting/rejecting. Not pre-specified. The antler is the only confirmed artifact (from the familiars). Everything else is to be generated.

3. **Specific holy artifact** — the item that unlocks the OOR perspective. Not yet designed. Needs to feel significant in the cosmology of Suboont.

4. **NPC/observer nodes** — nodes where the player has no choices and just witnesses. Implied by the Primordial Plane experience. Not yet formally in the schema as a `perspective` field value. Should add `observer` as a node-level perspective type.

5. **The failure state** — what actually happens when the portal doesn't close cleanly. Deliberately left ambient. May need one or two nodes that gesture at this without resolving it.

6. **Future event layering** — the architecture should support adding new branches, rituals, and artifact pools for future events without breaking existing playthroughs. This is a versioning question for `nodes.json`.

## Tone Touchstones

When in doubt, ask: does this read like a sentence from one of these?

- Annihilation (VanderMeer) — environmental uncanny, transformation without explanation
- The King in Yellow (Chambers) — known through effects, never directly
- Pan's Labyrinth — ambiguous ancient entities operating by pre-moral rules
- Invisible Cities (Calvino) — second-person address, layered fiction
- Late-night dance floor at 2am — the real referent underneath all of it
